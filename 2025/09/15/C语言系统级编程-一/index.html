C语言系统级课程(一)

作者：弎轮轟

一.从几个问题引入

1.变量

const关键字：用于声明一个只读变量，表示一旦被初始化就不可被改变(所以必须在初始化时赋初值)

volatile关键字：阻止编译器优化代码，因为所修饰的变量有可能被硬件或其他线程等未知方式改变，每次访问所修饰变量都需要从内存读取，而不是寄存器，举个例子：

  • 想象你在看一本电子书(变量)，你觉得内容不会变，就把关键内容记在脑子(寄存器缓存)里。但这本书是本 “活书”，可能被别人偷偷涂改(硬件 / 其他线程)
    • 没有volatile时：编译器会优化成 “直接用脑子里的内容”，即使书的内容已经变了，你看到的还是旧内容
    • 有volatile时：编译器会强制你 “每次都重新翻书看”，确保你总能拿到最新内容

对于以上几行代码，所谓“变量”值有时能改有时不能改，那么这时候称它们为“变量[Variable]“是否还恰当？

答案：标准文本并没有“变量[Variable]“这一概念

2.整数常量

10 是不是整数常量？ ——整数常量

+10 是不是整数常量？ ——常量表达式

(int)(10.0) 是不是整数常量？ ——常量表达式

(int)(+10.0) 是不是整数常量？ ——变长数组 (VLA) 的维度表达式

给定const int b=10; b 是不是整数常量？——不是

什么是整数常量？整数类型且值不会变的量? ——“编译器在编译阶段就能算出来的整型值”from ChatGPT

答案：只有10是整数常量，其他都是常量表达式

整数常量和整数常量表达式都是可以在编译期间计算得出的

3.编译期计算的sizeof()函数

现在i等于几？

答案：0，sizeof()函数在编译时就会计算结果，它不会执行操作数中的表达式i++，只会分析操作数的类型

现在i等于几？j等于几？

答案：i=0，j=1，(int)(10.0) 是整数常量表达式故而在编译期间可以确定第二维的大小，而(int)(+10.0)内部(+10.0)是浮点常量表达式，无法在编译期间确定第二维大小，所以需要再运行期间进行j++运算，导致j加一

二.C语言的一般化设计思想

1.一些C标准

对于第一部分的问题1，实际上的C标准中将a、b、c、d、e、f统称为对象标识符，其处理逻辑规则一致

为什么0[a]=1合法？

这里的0[a]=1是所谓的“语法糖”，因为数组访问本质上是一个加法：要访问a[i]，其实是"先计算指针a加上偏移量i，再解引用"，因此等价于*(a + i)，而C语言加法是commutative(可交换的)，故而又等价于*(i + a)，最后写成了i[a]，但是这种写法并不符合代码阅读习惯，反而会降低代码的可读性

  • 语法糖：某种语法的其他写法，这种写法对功能并没有影响，但是更方便程序员使用。通常来说使用语法糖能够增加程序的可读性，减少代码出错的机率

为什么foo()、(&foo)()和(*foo)()都合法？

C语言对函数名的特殊处理规则：函数名在表达式中会隐式转换为函数指针

根据上述特殊处理规则，我们可以对每条代码进行解读：

  • foo()经隐式转换后变为(&foo)()，然后通过函数指针访问函数
  • (&foo)()是显式获取函数foo的地址，无需多言
  • (*foo)()中，foo会先根据规则转换为&foo，由此原代码变为(*(&foo))()，而我们知道*和&操作互反，最终变为foo()，后续同第一条

2.学习资料与标准体系之间的差异

  • 核心概念介绍缺失：对象、lvalue(左值)、**rvalue(右值)**等核心概念介绍不详细或者不涉及

    • C语言中存在“对象“这一概念
    • lvalue(左值)：可寻址的对象
    • rvalue(右值)：没有持久的存储地址，或不能被取地址的临时值

  • 部分概念混淆有误：变量、常量的划分方法，常量和常量表达式的混淆等

  • 存在许多自创概念：常变量、只读变量、行指针、列指针、数组指针、指针数组、多级指针、模拟引用传递……

三.地址

  • 地址是字节在内存中的编号，只有字节才有地址，我们一般谈论地址的时候指的都是字节的逻辑地址

  • 1个byte不一定等于8个bit，<limits.h>中定义了一个宏CHAR_BIT，C语言中规定一个byte等于CHAR_BIT个bit，C语言标准规定CHAR_BIT>=8，即一个Byte包含至少8个bit

  • 我们所说的8/16/32/64(n)位机，是指用n位bit来表示byte的编号

四.对象

1.定义

在标准文档中有这样两句话：

region of data storage, the contents of which can represent values

一段数据存储区域，其内容能够表示数值

objects are composed of contiguous sequences of one or more bytes

对象由一个或多个字节组成的连续序列构成

说白了，C语言中的对象就是能表示有意义的值一段连续的内存区域

2.对象类型

简单说，对象类型就是 “给对象贴的标签”，告诉编译器：这块连续内存(对象)应该如何被理解、如何被操作

  • 完全对象类型 [Object Type] 对象的大小(size)、内存布局(layout)和对齐方式(alignment requirement)都很清楚的对象
  • 不完全对象类型 [incomplete Type] 与完全对象类型相反，包括对象信息不完整的对象类型和空类型(void)

3.对象值[Object Value]与对象表示[Object Representation]

  • 对象表示：指对象在内存中实际存储的二进制字节序列，是对象的 “物理形态”
  • 对象值：从对象表示中解读出的、有逻辑意义的数据，是对象的 “逻辑含义”

上述两者通过对象类型相互映射

假设有两个对象，他们的对象类型一样：

  • 若对象表示一样，则对象的值一定一样
  • 若对象的值一样，则对象表示不一定一样

原因：组成对象表示的bit中包括Value Bits(有效)和Padding Bits(填充)，而一个对象表示可以有若干无意义的Padding Bits，当然Value Bits和Padding Bits的构成也是由对象类型决定的

4.对象的size

对于任何一个对象来说，假设它包含的byte个数是n(n*≥1)

  • n是这个对象的size，这个对象包含的bit数量就是n×CHAR_BIT个

5.对象的地址和对齐要求

  • 对象的地址[address] 对象所占用的连续字节中最低位字节的编号
  • 对齐要求 [alignment] 要求对象的地址能被对齐要求整除

对于一个对象类型T，我们可以使用sizeof(T)获取类型大小(如果是不完全对象类型，就没有size)以及使用alignof(T)获取缺省对齐要求(对象的对齐要求可变)

  • 缺省：在特定情况下，某个设置或参数未被明确指定时所使用的默认值

类型本身没有大小和对齐要求，只有对象才有大小和对齐要求

五.类型

类型包括算数类型(Arithmetic Type)、派生类型(Derived Type)、限定类型(Qualified Type)

1.算数类型

还有一种分类Basic Type，包括char、Unsigned Integer Type、Signed Integer、Floating Type

  • char、signed char、unsigned char是三种不同的类型，编译器会指定char和signed char、unsigned char其中之一行为一致

  • 对于一个标准有符号整数类型来说，所有的bits分为三个部分：1个sign bit，n个value bits，m个padding bits，这时候这个对象类型的宽度为n+1，精度为n，对象值的范围是-(2^(n-1)到2^(n-1)-1

  • signed char的宽度一定等于CHAR_BIT，且signed char不允许有padding bits

  • 对于无符号整数类型(每种有符号整数类型对应一种无符号证书类型+bool类型)来说，所有的bits分为两个部分：n个value bits，m个padding bits，这时候这个对象类型的宽度为n，精度也为n，对象值的范围是0到2^n-1

  • unsigned char的宽度刚好等于CHAR_BIT

  • bool类型的宽度刚好为1(只需要1为表示true和false)，bool类型的size由编译器决定，且bool类型一定有padding bits，padding bits = sizeof(bool) * CHAR_BIT - 1

  • 对象的对齐要求一定是2^n

  • signed char/unsigned char类型的对齐要求是weakest，但不一定是1

  • 枚举类型enum Season是由程序员定义出来的

    • 在枚举类型定义的时候可以指定实现类型
    • 如果不指定实现类型，实现类型由编译器自行设置为以下5种整数类型之一：char、标准有符号整数类型、标准无符号整数类型、扩展有符号整数类型、以及扩展无符号整数类型

2.派生类型

(一)数组类型

数组类型的形式化定义：T[N]

  • T：元素类型[Element Type]，必须是一个完全对象类型
  • N：元素类型的个数[Number of Elements]，N是一个表达式
    • 若没有N，则T[]也是一个数组类型，且该数组类型是一个不完全对象类型
    • 若有N，如果N是一个整数常量或整数常量表达式，T[N]是普通数组类型
    • 否则，则T[N]是变长数组类型(Variable Length Array Type)

3.限定类型

(一)_Atomic

给定一个类型T，其原子限定类型是_Atomic(T)

  • _Atomic(T)是一个合法的新类型，_Atomic(T)和T是两种不同的类型

  • 他们的大小、对齐要求以及相同对象值对应的对象表示_不一定_一样

  • 该类型对象的大小和缺省对齐要求为：sizeof(_Atomic(T))和alignof(_Atomic(T))

  • 原子类型也是一种派生类型

(二)const、volatile、restrict

给定一个类型T，给定三个限定符之一的Q，其限定类型为Q T或T Q

  • T和Q T(或T Q)是两种不同的Type，但大小，表示值，对齐方式一样

  • restrict只能限定对象指针类型，或者对象指针类型构成的多维数组类型

  • 对于不同类型的T：

    • 当T是一个整体的时候，限定符在T的左边右边都一样
    • 当T[N]是数组类型的时候，Q T[N]：数组类型，元素类型为Q T，元素个数N；T[N] Q：不合法
    • 当T*是数组类型的时候，Q T*：指针类型，Referenced Type是Q T；T* Q：Q限定T*类型

4.为不同类型T设置别名

  • 语法上的整体：typedef T Alias
  • 数组类型：typedef T Alias[N]/T Alias [N]
  • 指针类型：typedef T* Alias/T *Alias/T*Alias/T * Alais
